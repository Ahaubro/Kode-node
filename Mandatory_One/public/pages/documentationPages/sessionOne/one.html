Rest API

There is some conventions when it comes to Rest Api. When you make an endpoint it is important to include these methods:

  • Get all - GET without ID
  • Find by id - GET with ID
  • Create new - POST without ID
  • Update ressource - PUT with ID
  • Update ressource partially- PATCH with ID

    • A RestApi should also meet the following conventions:

  • Ressource type should be a noun.
  • Endpoints ressource type should be a noun in plural.
  • Each ressource should have an id assignt when created
  • A ressource id should never be changed.
